> 编制日期：2026-05-14
> 说明：管理后台为独立 Web 前端项目，供甲方运营人员日常使用，与小程序并行开发，由前端工程师主导。

## 二、技术方案

### 2.1 技术选型

| 层面 | 选型 | 说明 |
|------|------|------|
| 框架 | **React 18 + Ant Design 5** | 国内最成熟的中后台方案，Table/Form/Chart 开箱即用 |
| 构建 | Vite 5 | 开发 HMR 快，生产构建小 |
| 状态管理 | Zustand / React Context | 轻量，不需要 Redux 的重量 |
| 图表 | ECharts 5 + echarts-for-react | 数据看板必备 |
| 表格 | Ant Design ProTable | 分页/筛选/排序/导出一体化 |
| 富文本 | Quill / TinyMCE | 专区内容编辑、协议模板编辑 |
| 上传 | Ant Design Upload + OSS 直传 | 图片/Banner/题目 Excel |
| 签名审核 | 图片预览组件 | 放大查看 canvas 签名图 |
| 权限 | 路由守卫 + RBAC | 管理员角色：超级管理员/内容编辑/客服/财务 |

### 2.2 后端 API 依赖

管理后台所有接口由后端模块 **B-23（管理后台 API）** 统一提供：

```
/admin/users          → 用户管理 CRUD
/admin/orders         → 订单管理 CRUD + 退款 + 对账
/admin/zones          → 专区内容 CRUD + Banner + 排序上下线
/admin/courses        → 课程 CRUD + 班次管理
/admin/quiz           → 题库 CRUD + 分类 + 批量导入
/admin/coupons        → 优惠券库管理 + 批量创建
/admin/agreements     → 协议模板 + 审核 + 盖章
/admin/conversations  → 对话记录查询 + 意图分析
/admin/tickets        → 工单处理 + 客服绩效
/admin/settings       → 系统配置 + 管理员 + 权限
/admin/statistics     → 数据看板聚合接口
/admin/competition    → 竞赛报名导出
```

**题目表单弹窗**：
- 所属分类（级联选择）
- 题型（单选/多选）
- 题目内容（textarea）
- 选项列表（动态添加选项 A/B/C/D）
- 正确选项（选择框）
- 答案解析（textarea，可选）

**Excel 批量导入流程**：
1. 下载模板（按分类预生成的 Excel 模板）
2. 上传文件（拖拽上传区域）
3. 格式校验（必填字段/选项格式/分类匹配）
4. 错误行提示（如有格式问题，标红显示）
5. 确认导入（预览前 10 条 + 总数统计）
6. 导入进度条（大批量异步处理）

- 富文本编辑器编辑协议正文

**签名审核列表**：
```
| 用户 | 协议类型 | 提交时间 | 状态 | 操作 |
|------|---------|---------|------|------|
| 张三 | 培训协议 | 6/10 14:30 | 待审核 | [查看签名] [通过] [驳回] |
```

- 盖章确认 → 状态变更为"已盖章"，用户端可查看
- 驳回 → 填写驳回原因，用户重新签名

## 七、部署方案

| 项目 | 方案 |
|------|------|
| 构建输出 | `dist/` 静态文件 |
| 部署方式 | Nginx 静态文件托管，与后端 API 同域名不同路径 |
| 路由策略 | `/admin` → 管理后台 SPA，`/api` → 后端 API |
| 域名 | 与后端 API 共用已备案域名，如 `admin.xxx.com` 或 `xxx.com/admin` |
| CI/CD | 与前端小程序共用 CI 流水线，独立部署步骤 |
| 权限 | Nginx 层 IP 白名单（可选）+ JWT 登录态 |

## 八、风险管理

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| B-23 管理 API 未按时交付 | 中 | 后台页面无法联调 | 前端先用 Mock 数据开发页面，API 就绪后切换 |
| 题库 Excel 格式多样 | 高 | 导入失败率高 | 提供严格格式模板 + 导入前预校验 + 详细错误提示 |
| 数据看板性能（大数据量时） | 低 | 页面卡顿 | 后端聚合数据 + 前端 ECharts 懒加载 + 时间范围默认近30天 |
| 甲方管理员上手困难 | 中 | 无法独立运营 | 交付时提供操作手册（含截图步骤）+ 2h 培训 |
