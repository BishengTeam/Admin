# 后端对接执行步骤

> 基于 `docs/前后端对接决策表.md`，后端改动全部在此。
> 原则：不改现有小程序 schema，管理后台功能补齐。

---

## P0 — 登录返回权限 + 认证联调

- [ ] **#2 登录接口返回 permissions**
  - `app/services/admin_auth.py` `AdminAuthService.login()` 中返回 `data.permissions`
  - 权限码列表与前端 `src/routes.tsx` 中的 `meta.permission` 对齐
  - 示例：
    ```python
    ALL_PERMISSIONS = [
        "dashboard:view", "user:list",
        "order:list", "order:write",
        "quiz:list", "quiz:write", "quiz:import",
        "content:list", "content:write", "content:banner",
        "course:list", "course:write",
    ]
    ```

- [ ] **#2 `AdminLoginResponse` schema 增加字段**
  - `app/schemas/admin.py` `AdminLoginResponse` 增加 `permissions: list[str]`
  - 无需新增数据库表，固定返回全权限数组即可

## P1 — 补齐缺失的管理接口

- [ ] **#5.1 用户批量删除**
  - 新增 `POST /admin/users/batch-delete`
  - `app/api/admin/users.py` 新增路由
  - `app/services/admin_user.py` 新增 `batch_delete(user_ids: list[int])` 方法
  - 请求体：`{ "ids": [1, 2, 3] }`
  - 注意事项：仅标记 `is_deleted`（软删除），不物理删除

- [ ] **#5.2 题目批量删除**
  - 新增 `POST /admin/quiz/questions/batch-delete`
  - `app/api/admin/quiz.py` 新增路由
  - `app/services/admin_quiz.py` 新增 `batch_delete_questions(question_ids: list[int])` 方法
  - 请求体：`{ "ids": [1, 2, 3] }`

- [ ] **#5.3 内容上下架切换**
  - 新增 `PATCH /admin/zones/{zone_id}/status`
  - `app/api/admin/zones.py` 新增路由
  - `app/services/admin_zone.py` 新增 `toggle_status(zone_id, is_active)` 方法
  - 或直接复用 `PUT /admin/zones/{zone_id}` 的 `is_active` 字段，无需单独端点

- [ ] **#5.4 内容批量删除（下架）**
  - 新增 `POST /admin/zones/batch-delete`
  - `app/api/admin/zones.py` 新增路由
  - `app/services/admin_zone.py` 新增 `batch_deactivate(zone_ids)` 方法
  - 请求体：`{ "ids": [1, 2, 3] }`

- [ ] **#5.5 Banner 管理 CRUD**
  - 新建 `app/models/banner.py` — 数据表：`id`, `image_url`, `jump_link`, `sort`, `start_time`, `end_time`, `is_active`, `created_at`
  - 新建 `app/schemas/admin_banner.py` — `BannerCreate`, `BannerUpdate`, `BannerListItem`
  - 新建 `app/services/admin_banner.py` — CRUD 方法
  - 新建 `app/api/admin/banners.py` — 路由注册
  - `app/api/admin/__init__.py` — 注册 banner router
  - Alembic 迁移 — 新增 banner 表
  - 接口：
    - `GET /admin/banners` — 全量列表（不分页）
    - `POST /admin/banners` — 创建
    - `PUT /admin/banners/{id}` — 编辑
    - `DELETE /admin/banners/{id}` — 删除
    - `POST /admin/banners/batch-delete` — 批量删除

- [ ] **#6 题库 JSON 批量导入**
  - `app/api/admin/quiz.py` 新增 `POST /admin/quiz/import/json` 路由
  - 请求体结构：
    ```json
    {
      "category_id": 1,
      "questions": [
        {
          "question_text": "题目内容",
          "options": {"A": "选项A", "B": "选项B", "C": "选项C", "D": "选项D"},
          "correct_answer": "C",
          "question_type": "single"
        }
      ]
    }
    ```
  - `app/services/admin_quiz.py` 新增 `import_questions_json(data)` 方法
  - 保留原有 CSV 导入接口不变

## P2 — 补齐筛选 + 导出 + 详情子查询

- [ ] **#7.1 用户列表筛选增强**
  - `app/api/admin/users.py` `list_users` 增加 query 参数：`openid`（已有），新增 `created_at_start`、`created_at_end`
  - `app/schemas/admin.py` `AdminUserFilter` 增加对应字段
  - `app/services/admin_user.py` `list_users` 增加时间范围 WHERE 条件
  - 注意：`openid` 模糊搜索还是精确匹配？当前是精确匹配

- [ ] **#7.2 订单列表筛选增强**
  - `app/api/admin/orders.py` `list_orders` 增加 query 参数：`cert_type`、`phone`
  - `app/schemas/order.py` `OrderFilter` 增加对应字段
  - `app/services/admin_order.py` `list_orders` 增加 WHERE 条件

- [ ] **#7.3 课程列表筛选增强**
  - `app/api/admin/courses.py` `list_courses` 增加 query 参数：`keyword`（title ILIKE）、`category`
  - `app/services/admin_course.py` `list_courses` 增加 WHERE 条件

- [ ] **#7.4 内容列表筛选增强**
  - `app/api/admin/zones.py` `list_zones` 增加 query 参数：`keyword`（title ILIKE）
  - `app/services/admin_zone.py` `list_zones` 增加 WHERE 条件

- [ ] **#5.6 用户导出**
  - 新增 `GET /admin/users/export`
  - `app/api/admin/users.py` 新增路由
  - `app/services/admin_user.py` 新增 `export_users(filters)` 方法
  - 返回 CSV 文件流（`Response(media_type="text/csv")`）

- [ ] **#5.7 用户详情-订单子查询**
  - 新增 `GET /admin/users/{user_id}/orders`
  - `app/api/admin/users.py` 新增路由
  - `app/services/admin_user.py` 新增 `get_user_orders(user_id)` 方法
  - 返回该用户的所有订单摘要列表

- [ ] **#5.8 用户详情-对话记录子查询**
  - 新增 `GET /admin/users/{user_id}/conversations`
  - `app/api/admin/users.py` 新增路由
  - `app/services/admin_user.py` 新增 `get_user_conversations(user_id)` 方法
  - 返回该用户的 AI 对话记录列表

- [ ] **#5.9 订单导出**
  - 新增 `GET /admin/orders/export`
  - `app/api/admin/orders.py` 新增路由
  - `app/services/admin_order.py` 新增 `export_orders(filters)` 方法
  - 返回 CSV/XLSX 文件流

- [ ] **#5.10 对账数据**
  - 新增 `GET /admin/orders/reconciliation`
  - `app/api/admin/orders.py` 新增路由
  - query 参数：`date`（YYYY-MM-DD）
  - 返回：`{ order_total, refund_total, net_income, order_count }`

- [ ] **#5.11 内容拖拽排序**
  - 新增 `PUT /admin/zones/sort`
  - `app/api/admin/zones.py` 新增路由
  - 请求体：`[{ "id": 1, "sort_order": 0 }, { "id": 2, "sort_order": 1 }]`
  - `app/services/admin_zone.py` 新增 `update_sort(updates)` 方法
  - 建议用单个 SQL UPDATE 批量更新，避免逐条 UPDATE
