# 手工测试链路

> 编制日期：2026-06-01  
> 项目：小程序平台管理后台

---

## 链路 1：登录 → 导航 → 注销

| 步骤 | 操作 | 预期 |
|------|------|------|
| 1.1 | 浏览器打开 `/admin` | 跳转 `/admin/login` |
| 1.2 | 输入用户名 `admin`、密码 `admin123`，点击登录 | 跳转 `/admin/dashboard` |
| 1.3 | 检查左侧菜单 | 显示 7 项：数据看板、用户管理、订单管理、题库管理、内容管理、Banner配置、课程管理 |
| 1.4 | 点击「用户管理」 | 跳转 `/admin/users`，表格加载用户列表 |
| 1.5 | 点击「订单管理」 | 跳转 `/admin/orders`，表格加载订单列表 |
| 1.6 | 点击「内容管理」 | 跳转 `/admin/content`，子菜单展开 |
| 1.7 | 点击右上角用户名 → 「退出登录」 | 弹出确认，确认后跳转 `/admin/login`，菜单不再可见 |

## 链路 2：数据看板

| 步骤 | 操作 | 预期 |
|------|------|------|
| 2.1 | 登录后进入 `/admin/dashboard` | 6 个指标卡显示真实数值（非 0），含「注册用户数」「订单总数」「近30日订单」「已支付订单」「近30日营收 ¥xxx.xx」「付费转化率 xx%」 |
| 2.2 | 刷新页面 (F5) | 指标卡数值保持，不闪烁 0 |

## 链路 3：用户管理 CRUD + ConfirmButton

| 步骤 | 操作 | 预期 |
|------|------|------|
| 3.1 | 进入 `/admin/users` | 表格加载用户列表 |
| 3.2 | 点击某行「查看」 | 右侧弹出用户详情抽屉 |
| 3.3 | 点击某行「禁用」 | 弹出 ConfirmButton 确认框「确认禁用此用户？」，确认后状态切换为已禁用 |
| 3.4 | 点击某行「删除」 | 弹出 ConfirmButton 确认框「此操作不可撤销，确认删除此用户？」，确认后用户从列表消失 |
| 3.5 | 勾选多行，点击「批量删除」 | 弹出确认框，确认后选中用户全部删除 |
| 3.6 | 点击「导出」 | 下载用户数据 CSV 文件 |

## 链路 4：内容管理 + Banner + 课程 CRUD

| 步骤 | 操作 | 预期 |
|------|------|------|
| 4.1 | 进入 `/admin/content` | 表格加载内容列表，支持搜索、新增、编辑 |
| 4.2 | 点击某行「删除」 | ConfirmButton 确认后删除 |
| 4.3 | 勾选多行，批量删除 | ConfirmButton 确认后批量删除 |
| 4.4 | 进入 `/admin/content/banners` | Banner 列表，支持新增/编辑/删除/批量删除，均使用 ConfirmButton |
| 4.5 | 进入 `/admin/content/courses` | 课程列表，支持搜索/新增/编辑/删除，删除使用 ConfirmButton |

## 链路 5：题库管理

| 步骤 | 操作 | 预期 |
|------|------|------|
| 5.1 | 进入 `/admin/quiz` | 左侧分类树 + 右侧题目列表正常加载 |
| 5.2 | 点击分类节点 | 题目列表按分类筛选 |
| 5.3 | 删除分类 | ConfirmButton「删除此分类将同时删除其子分类和题目」 |
| 5.4 | 删除题目 | ConfirmButton 确认后删除 |
| 5.5 | 批量删除题目 | ConfirmButton 确认后批量删除 |
| 5.6 | 点击「批量导入」 | 跳转 `/admin/quiz/import` |

## 链路 6：Token 过期 + AuthGuard

| 步骤 | 操作 | 预期 |
|------|------|------|
| 6.1 | 登录后在 DevTools 执行 `localStorage.setItem('admin_token', 'eyJhbGciOiJIUzI1NiJ9.eyJleHAiOjEwMDAwMDAwMDB9.xxx')` | — |
| 6.2 | 刷新页面 | 立即跳转 `/admin/login`（不闪现管理页面） |
| 6.3 | 重新正常登录 | 回到 `/admin/dashboard` |

## 链路 7：页面级 ErrorBoundary 隔离

| 步骤 | 操作 | 预期 |
|------|------|------|
| 7.1 | 后端返回异常数据导致渲染崩溃 | 页面显示「页面加载失败」+「重试」按钮，顶部导航和侧边栏仍正常可用 |
| 7.2 | 点击「重试」 | 组件重新挂载，重新请求数据 |
| 7.3 | 点击其他菜单 | 正常跳转，不受崩溃页面影响 |

## 链路 8：权限校验（非 admin 用户）

| 步骤 | 操作 | 预期 |
|------|------|------|
| 8.1 | 用仅有部分权限的账号登录（如 `permissions: ["user:list", "order:list"]`） | 菜单仅显示「用户管理」「订单管理」 |
| 8.2 | 直接访问 `/admin/dashboard` | 能打开页面但后端返回 403 或数据异常 |
| 8.3 | 用 `permissions: ["*"]` 的 admin 登录 | 全部 7 项菜单可见 |

## 链路 9：浏览器刷新持久化

| 步骤 | 操作 | 预期 |
|------|------|------|
| 9.1 | 登录后在任意页面按 F5 | 页面重新加载，`/admin/auth/me` 失败时降级保留 token，菜单全显 |
| 9.2 | 刷新后点击菜单项 | 正常跳转，API 携带 Bearer token |
| 9.3 | 长期闲置后 token 过期，刷新 | 跳转 `/admin/login` |
